JVM(Java Virtual Machine)이란?
바이트코드(.class)를 실행해 애플리케이션이 운영체제·cpu에 상관없이 
동일하게 동작하도록 하는 실행 환경을 제공하는 것이다.

쉽게 말해서,

Java 프로그램이 Windows, macOS, Linux 에서 똑같이 실행되게 만든다.

즉, JVM 덕분에 Java는 한번 작성하면 
어디서든 실행(Write Once, Run Anywhere)이 가능하다.

JVM은 단순히 .class 파일을 실행하는 것 이상을 한다.

JVM 3가지 구성 요소
1. Class Loader (클래스 로더)

- .class 파일을 런타임에 메모리에 적재


- 필요한 순간에만 동적으로 클래스 로딩
? 약간 모호함

- 같은 클래스를 중복 로딩하지 않도록 관리


2. Runtime Data Area (메모리)

JVM 실행 시 사용하는 메모리 공간

영역 	용도
Method Area	클래스 정보, static 변수, 메서드 코드 저장
Heap	객체가 저장되는 영역 (GC의 대상)
Stack	메서드 호출, 지역 변수, return 정보
PC Register	현재 실행 중인 JVM 명령어의 주소를 저장
Native Method Stack	JNI로 호출되는 C/C++ 코드 실행 공간
JVM이 실행되면 내부에서 5개의 Runtime Data Area(메모리 영역) 이 생성되고, 이 메모리 구조가 동작하면서 Java 프로그램이 실행될 수 있도록 지원한다.



3. Execution Engine (실행 엔진)

- 인터프리터: 바이트코드를 한 줄씩 해석

- JIT(Just-In-Time) 컴파일러: 자주 실행되는 코드를 기계어로 즉시 컴파일

- GC(Garbage Collector): 사용되지 않는 객체 자동 제거



JVM 작동 순서(실행 원리)

.java  →  (javac 컴파일)
.class(바이트코드)
      ↓
[Class Loader] : 클래스 로딩
      ↓
[Bytecode Verifier] : 보안 검사
      ↓
[Execution Engine] : 인터프리트 + JIT 실행
      ↓
[Runtime Data Area] : 메모리 (Heap/Stack/Method Area 등)
      ↓
[Garbage Collector] : 비사용 객체 회수
 

JVM / JRE / JDK 차이


JVM : 바이트코드(.class)를 기계어로 변환하여 Java 프로그램을 실제로 실행하는 엔진

JRE : Java 프로그램이 실행될 수 있도록 JVM + 필수 라이브러리(클래스 API) 를 제공하는 실행 환경

JDK : Java 프로그램을 개발하기 위한 도구 세트 → JRE + 컴파일러(javac) + 개발 도구

즉,
JVM = 실행 엔진
JRE = 실행 엔진 + 실행에 필요한 라이브러리
JDK = 실행 + 개발까지 가능한 전체 도구 세트 이다.
